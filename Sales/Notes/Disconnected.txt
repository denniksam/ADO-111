Отсоединенный режим работы с БД

Присоединенный режим - постоянно открытое подключение (программы к БД),
 каждый запрос обращается непосредственно к БД

Отсоединенный режим - "импульсное" взаимодействие с БД:
 - при старте программы данные загружатся в программу (в массивы/коллекции)
 - запросы на выборку данных направлены на загруженные данные
 - в базу данных запросы отправляются либо при закрытии программы, либо
    с некоторой периодичностью для синхронизации данных

Отсоединенный режим значительно быстрее, но может приводить к неактуальности
данных (из-за редкой синхронизации). Этот режим хорош тогда, когда у БД
один клиент (одна программа работает с БД) либо когда данные изменяются редко,
в основном "на чтение"

Присоединенный режим используется в многопользовательском режиме или при
частых изменениях данных (сайты, мессенджеры)

Для работы с загруженными данными популярна технология (паттерн) MVVM
Его особенность - связывание представления с данными. Для этого в WPF
используются специальные типы, например, ObservableCollection
Изменения в этой коллекции приводят к появлению внутренних событий, которые
обновляют представление (! коллекция должна быть public для доступности
из представления)
